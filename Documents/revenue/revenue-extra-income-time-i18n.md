# 系统概述
本需求针对收入管理模块的“额外收入列表”，将“录入时间”字段的展示改为国际化显示，随前端语言切换。

# 架构设计
- 模块：ChatUI（收入管理页面）
- 依赖：LanguageContext（前端语言上下文）
- 格式化：使用统一的日期格式化工具（优先使用现有工具或 `Intl.DateTimeFormat`）

# 功能特性
- 额外收入列表中“录入时间”字段需支持中英文切换显示。
- 切换语言后无需刷新页面即可更新时间格式。
- 时间显示不改变业务含义，只改变展示格式。

# 技术实现
- 使用 LanguageContext 获取当前语言。
- 统一在列表渲染时对 `createdAt` / `createdTime` 字段做格式化。
- 若存在通用时间格式化工具，优先复用；否则使用 `Intl.DateTimeFormat`。
- 建议格式（可根据现有规范调整）：
  - 中文：`YYYY-MM-DD HH:mm`
  - 英文：`MMM DD, YYYY HH:mm`

# API 接口文档
- 无新增或修改接口。

# 数据模型
- 额外收入记录包含时间字段（如 `createdAt`），用于展示。

# 使用指南
- 在收入管理的额外收入列表中查看“录入时间”，会随语言切换自动变化格式。
