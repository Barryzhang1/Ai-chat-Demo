# 系统概述
本需求用于座位管理功能的前端提示词国际化，覆盖餐厅开门/关门后的弹出消息，确保多语言环境下提示内容正确显示。

# 架构设计
- 影响范围：ChatUI（前端）
- 入口页面：座位管理页面
- 国际化来源：前端已有 LanguageContext 与本地化文案存储

# 功能特性
- 在座位管理页面点击开门/关门后，弹出消息根据当前语言显示对应文案。
- 默认语言为中文，切换语言后提示文案同步切换。

# 技术实现
- 定位座位管理页面中开门/关门操作后的弹窗或 Toast 文案来源。
- 使用既有的语言上下文（LanguageContext）或本地化字典进行文案映射。
- 保持现有交互流程不变，仅替换提示文案来源。

# 实现说明
- 已在前端座位管理页的开门/关门 Toast 中使用本地化文案作为默认提示。
- 当后端返回中文提示时，中文环境继续显示原消息；英文环境统一显示英文文案。

# API 接口文档

- 无新增数据模型。

# 使用指南
1. 进入座位管理页面。
2. 点击开门或关门。
3. 验证弹出提示在不同语言下显示正确文案。
